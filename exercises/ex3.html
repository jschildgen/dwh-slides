Exercise 1: Multi-Version Concurrency Control

When you solve this exercise on your local database, create a new schema for this exercise: CREATE SCHEMA IF NOT EXISTS ex3; If you use the OTH database (194.95.108.54), a user is created for every student. As username, password and schema name use upper(replace(substr(email, 0, instr(email, '@')-1),'.','_')). Example: When your OTH email address is firstname.lastname@oth-regensburg.de, your username, password, and schema name is FIRSTNAME_LASTNAME.

  1. Create a table SETTINGS with two VARCHAR(2000000) columns K and V (key and value).
  2. Insert a row ('GREETING', 'Hello') in the settings table.
  3. In your DBeaver settings (Window/Fenster, Settings/Einstellungen), under Editors, SQL-Editor enable "Open a separate connection for each editor" This means, when you open multiple tabs, they will not use the same session but open multiple sessions.
  4. Open two editor tabs and check whether they use different sessions: SELECT CURRENT_SESSION;
  5. Disable autocommit (Database, Transaction Mode, Manual Commit) in both sessions.
  6. Commit in both editors to start new transactions.
  7. In one transaction, update the greeting to 'Hi', in the other one insert a new row in the table. Then commit both transactions. Was there a waiting situation?
  8. In both transactions, SELECT * FROM settings;, and then, again, perform an update in one and an insert in the other transaction. The second transaction will be rolled back. Why?
