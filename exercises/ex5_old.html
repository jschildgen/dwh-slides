Exercise 1: Data Modelling

When you solve this exercise on your local database, create a new schema for this exercise: CREATE SCHEMA IF NOT EXISTS ex5; If you use the OTH database, use your FIRSTNAME_LASTNAME user and schema.

In this exercise, we are planning a data warehouse schema for a bike-rental company.
They will load customer data from a relational database. Each customer has a cust_id, name, email, phone, street, area, city, plan, price.
Furthermore, bikes are imported from Excel files which are provided by the different bike manufacturers. Each bike has bike_id, bike_type, model, brand.
The company has different bike stations which have a station_id, street, area, city
The bike-rental events can be loaded via an API. This API provides the information which customers drove which bike when from which station to which station. It shows the timestamp of issue, the rental duration, the rental price, and the number of bikes that parked at the return station after the rental.

  1. Draw an ME/R Model for this scenario. Which dimensions are hierarchical?
  2. For each measure, which data type, range, and aggregation type do they have?
  3. Map your model to a relational star schema by writing CREATE TABLE commands to create a star schema in your database. Which ones are the dimension tables, which one is the fact table?
  4. Insert two example rows in each of your tables.
  5. Transform your star schema (including the data) into a snowflake schema with CREATE TABLE and INSERT INTO ... SELECT commands. Use a prefix sf_ for the new tables (e.g., sf_markets). You can use the IDENTITY option to automatically generate unique primary-key values: my_id INT INDENTITY
