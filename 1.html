OLTP vs. OLAP

OLTP (OnLine Transaction Processing)

  • simple queries with short response time (SELECT, INSERT, UPDATE, ...)
  • small transactions
  • high concurrency (many users)
SELECT description, price FROM products WHERE product_no = 88;
SELECT P.description, AVG(R.rating)
FROM products P JOIN ratings R ON P.product_no = R.product_no 
WHERE P.product_no = 88;
INSERT INTO orders(order_id, customer_id) VALUES (101, 5);
INSERT INTO orders_positions(order_id, product_no, amount) 
       VALUES (101, 88, 1);
COMMIT;
In OLTP applications, very many short and simple transactions read and write large amounts of data.

Data Warehouse

A Data Warehouse is a physical database which allows an integrated view to arbitrary data for analysing them.

  • One large repository for the entire company
  • Dedicated hard- and software

Definition by Bill Inmon

A Data Warehouse is a subject-oriented, integrated, non-volatile, and time-variant collection of data in support of management decisions

Which data should be included in a backup?

  • Only the base data in the data sources
  • + the Raw Data in the data warehouse
  • + the Summary Data
  • Everything

https://fraage.de

Data is extracted from the data sources (DBs, files, APIs, ...) and stored into a staging area for further processing, cleaning and transformation. Next, the data is loaded into the data warehouse, together with its metadata (origin, timestamps, ...). Summary data is pre-aggregated data that allows faster analytics. Different users access the same data via different data marts. A data mart contains a subset of the data warehouse for specific needs.

Architecture Alternatives

Variants of the full data warehouse architecture:

Independent Data Marts

  • Data is directly loaded from the sources into the data marts
  • Multiple "mini data warehouses"
  • No single point of truth

Virtual Data Marts

  • Data marts as (logical) views on the data warehouse

What's the main benefit of materialized (non-virtual) data marts?

  • Faster access
  • Less disk space
  • Less RAM usage
  • Always up-to-date

What is Change-Data-Capture?

  • Keeping a history of data in the DWH
  • Mechanism to avoid transaction conflicts
  • Detect how data changed since last ETL run
  • Propagate changes from the DWH back to the data sources
